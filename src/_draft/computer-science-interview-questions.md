## Object-Oriented Programming

1. Encapsulation = hiding data by defining accessors and mutators.
2. Polymorphism = one name, many forms.
3. Inheritance = relation between objects that are either "has-a" or "is-a".
4. Abstraction = defines the essential characteristics of an object that defines it from all other kinds of objects, and thus provide crisply defined conceptual boundaries, relative to the perspective of the viewer" (Booch)

Single Responsibility, Open-Closed, Liskov Substitution, Interface Segregation, Dependency Injection (SOLID)

Michael Feather's "five principles of OOP programming and design." (Wikipedia)
